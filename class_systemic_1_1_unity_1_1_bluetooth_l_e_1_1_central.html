Detailed Description

A static class with methods for discovering, connecting to, and interacting with Bluetooth Low Energy (BLE) peripherals.

Use the ScanForPeripheralsWithServices method to discover available BLE peripherals. Then connect to a scanned peripheral with a call to ConnectPeripheralAsync. Once connected, the peripheral can be queried for its name, MTU, RSSI, services and characteristics. Characteristics can be read, written and subscribed to.

  • Be sure to disconnect the peripheral once it is not needed anymore.
  • Calls from any thread other than the main thread throw an exception.
  • Any method ending by Async returns an enumerator which is meant to be run as a coroutine.
  • A GameObject named SystemicBleCentral is created upon calling Initialize and destroyed on calling Shutdown.

This class leverages NativeInterface to perform most of its operations.

Here is a simplified example for scanning, connecting and sending a message to a peripheral.

// Scan for all peripherals (it's best to specify which services are required)
Central.ScanForPeripheralsWithServices();
// Wait until we get at least one peripheral
yield return new WaitUntil(() => Central.ScannedPeripherals.Length > 0);
// Stop scanning (saves battery life on mobile devices)
Central.StopScan();
// Select first peripheral
var peripheral = Central.ScannedPeripherals[0];
// And attempt to connect to it
var request = Central.ConnectPeripheralAsync(peripheral, (_, connected)
=> Debug.Log(connected ? "Connected!" : "Not connected!"));
yield return request;
// Check result
if (request.IsSuccess)
{
// And send some data
yield return Central.WriteCharacteristicAsync(
peripheral, aServiceUuid, aCharacteristicUuid, anArrayOfBytes);
}

Asynchronously connects to a discovered peripheral.

The enumerator stops on either of these conditions:

  • The connection succeeded.
  • DisconnectPeripheralAsync was called for the given peripheral.
  • The connection didn't succeeded after the given timeout value.
  • An error occurred while trying to connect.

Once connected to the peripheral, Central sends a request to change the peripheral's Maximum Transmission Unit (MTU) to the highest supported value.

Once the MTU is changed, Central notifies the caller that the peripheral is ready to be used by invoking the onConnectionEvent handler with the second argument set to true.

Upon a disconnection (whichever the cause), Central notifies the caller by invoking the onConnectionEvent handler with the second argument set to false.

Check RequestEnumerator members for more details.

Parameters
peripheralScanned peripheral to connect to.
onConnectionEventCalled each time the connection state changes, the peripheral is passed as the first argument and the connection state as the second argument (true means connected).
timeoutSecThe timeout value, in seconds. The default is zero in which case the request never times out.
Returns
An enumerator meant to be run as a coroutine. See RequestEnumerator properties to get the request status.

Gets the Maximum Transmission Unit (MTU) for the given peripheral.

The MTU is the maximum length of a packet that can be send to the BLE peripheral. However the BLE protocol uses 3 bytes, so the maximum data size that can be given to WriteCharacteristicAsync is 3 bytes less than the MTU.

Property Documentation

◆ IsReady

bool Systemic.Unity.BluetoothLE.Central.IsReady
staticget

Indicates whether Central is ready for scanning and connecting to peripherals. Reasons for not being ready are:

  • Initialize hasn't been called, or Shutdown was called afterwards.
  • Initialization is still on-going.
  • The host device either doesn't have a Bluetooth radio, doesn't have permission to use the radio or it's radio is turned off.

Detailed Description

Static class that scan for and connect to Pixel dice. Scan and connection requests are counted, meaning that the same number of respectively scan cancellation and disconnection requests must be made for them to effectively happen.

This allows for different parts of the user code to work with this singleton without impacting each others.

Requests to connect to the given Pixel.

Each Pixel object maintains a connection counter which is incremented for each connection request. and decremented for each disconnection request. The same number of disconnection requests than connection requests must be made to disconnect the Pixel.

This allows for different parts of the user code to request a connection or a disconnection without impacting each others.

Parameters
pixelThe Pixel to connect to.
requestCancelFuncA callback which is called for each frame during connection, it may return true to cancel the connection request.
onResultAn optional callback that is called when the operation completes successfully (true) or not (false) with an error message.
connectionTimeoutThe connection timeout in seconds.
Returns
The coroutine running the request.
